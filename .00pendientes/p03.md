# Impacto y Riesgos

El cambio propuesto **no debe tocar la lógica del backend ni el modelo de datos**. Solo agrega el frontend Nuxt al entorno Docker local y reemplaza el script que hoy abre Flutter web por uno que abra Nuxt.

Riesgos controlados:

1. Si se reemplaza completo `infra/docker-compose.yml`, se puede romper la configuración actual de backend/Postgres.
2. Si se usa `localhost:8000` dentro del contenedor frontend, el navegador SSR del contenedor no verá el backend. Dentro de Docker debe usarse el nombre del servicio backend.
3. Si se instala el frontend sin volumen de `node_modules`, el hot reload puede comportarse mal.
4. Si el backend aún no está migrado o sembrado, las pantallas abrirán pero varias acciones fallarán.

Protección aplicada:

* El cambio será **aditivo**: solo se agrega servicio `web_frontend`.
* No se modifican rutas API existentes.
* Se conserva el backend actual.
* Se deja rollback en menos de 2 minutos.

# Implementación Paso a Paso

## 1) Crear `web_frontend/Dockerfile.dev`

**Ruta exacta:** `web_frontend/Dockerfile.dev`

```dockerfile
FROM node:20-alpine

WORKDIR /app

# ===== NUEVO: dependencias del frontend =====
COPY package*.json ./
RUN npm install

# ===== NUEVO: código fuente =====
COPY . .

EXPOSE 3000

# ===== NUEVO: Nuxt en modo desarrollo accesible desde Docker =====
CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0", "--port", "3000"]
```

---

## 2) Agregar servicio `web_frontend` en `infra/docker-compose.yml`

**Importante:** no borrar lo existente.
Solo agregar este bloque dentro de `services:` y, si no existe, agregar también el volumen `web_frontend_node_modules:` al final.

### Bloque de servicio a pegar

```yaml
  web_frontend:
    build:
      context: ../web_frontend
      dockerfile: Dockerfile.dev
    container_name: buscamedicos_web_frontend
    environment:
      NUXT_PUBLIC_API_BASE: http://backend:8000/api/v1
      HOST: 0.0.0.0
      PORT: 3000
      NODE_ENV: development
    ports:
      - "3000:3000"
    volumes:
      - ../web_frontend:/app
      - web_frontend_node_modules:/app/node_modules
    depends_on:
      - backend
    restart: unless-stopped
```

### Volumen a pegar al final del archivo

```yaml
volumes:
  web_frontend_node_modules:
```

## Muy importante sobre este compose

Si el archivo actual ya tiene una sección `volumes:`, no duplicarla.
Solo agregar:

```yaml
  web_frontend_node_modules:
```

dentro de la sección ya existente.

Si el servicio backend en el compose **no se llama `backend`**, entonces en esta línea:

```yaml
NUXT_PUBLIC_API_BASE: http://backend:8000/api/v1
```

se debe reemplazar `backend` por el nombre real del servicio FastAPI.

## 3) Reemplazar `infra/scripts/dev_up_web.sh`

En el repositorio actual ese script llama a Docker y luego intenta abrir Flutter web desde `app_flutter` .
Se debe dejar así:

**Ruta exacta:** `infra/scripts/dev_up_web.sh`

```sh
#!/usr/bin/env sh
set -e

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"

cd "$REPO_ROOT/infra"
docker compose up -d --build

echo ""
echo "Docker services are up."
echo "Backend:  http://localhost:8000"
echo "Frontend: http://localhost:3000"
echo ""
echo "Para ver logs:"
echo "  docker logs buscamedicos_backend --tail 100"
echo "  docker logs buscamedicos_web_frontend --tail 100"
```

Dar permisos:

```bash
chmod +x infra/scripts/dev_up_web.sh
```
