Se deja el bloque completo, listo para pasar al programador, para que el flujo del médico quede funcional en local con Docker, con CRUD de documentos, carga real del PDF, validación manual posterior por un administrador y activación solo cuando la verificación quede aprobada. El propio repositorio ya define ese flujo esperado para profesionales y administración de validación, incluyendo `/professionals/me/documents`, `/professionals/me/submit-verification` y los endpoints administrativos de revisión. 

## 1) Impacto y riesgos

**Qué está mal ahora mismo**

El código actual de carga de documentos del profesional solo lee el archivo en memoria, calcula el hash y crea un `ProfessionalDocument`, pero no persiste el archivo físicamente ni crea el registro en `files`, por lo que el “CRUD” queda incompleto y no hay evidencia descargable para que validación la revise después.

**Qué se protege con este cambio**

Se protege que:

* el médico no quede activo ni público sin cargar el PDF y sin aprobación administrativa, como ya lo exige la regla de negocio del proyecto; 
* el PDF sí quede guardado en disco local dentro del contenedor y asociado a metadatos;
* el administrador pueda aprobar o rechazar documento por documento;
* el estado final del profesional solo pase a activo cuando la solicitud quede aprobada;
* el flujo siga funcionando en Docker local usando el volumen de archivos ya existente en `infra/docker-compose.yml`. 

**Riesgos a vigilar**

* El helper actual `audit_log()` en `admin.py` hace `commit()` interno y puede fragmentar transacciones.
* `useApi.ts` en frontend suele romper cargas `multipart/form-data` si se fuerza `Content-Type: application/json`.
* Si se aprueba una solicitud sin aprobar antes el PDF del título, el sistema podría dejar inconsistencia funcional.

```python
import hashlib
import os
import shutil
import uuid
from pathlib import Path
from typing import Tuple

from fastapi import HTTPException, UploadFile, status

from app.core.config import settings


class LocalFileStorageService:
    @staticmethod
    def _ensure_base_path() -> Path:
        base = Path(settings.FILES_PATH).resolve()
        base.mkdir(parents=True, exist_ok=True)
        return base

    @staticmethod
    def _validate_file(upload: UploadFile, size_bytes: int, force_pdf: bool = False) -> None:
        filename = (upload.filename or "").strip()
        ext = Path(filename).suffix.lower()
        mime = (upload.content_type or "application/octet-stream").lower()

        if not filename:
            raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail="El archivo no tiene nombre")

        allowed_mimes = settings.allowed_document_mime_types_list
        allowed_exts = settings.allowed_document_extensions_list

        if force_pdf:
            if ext != ".pdf" or mime != "application/pdf":
                raise HTTPException(
                    status_code=status.HTTP_400_BAD_REQUEST,
                    detail="Para este tipo de documento solo se permite PDF"
                )
        else:
            if ext not in allowed_exts:
                raise HTTPException(
                    status_code=status.HTTP_400_BAD_REQUEST,
                    detail=f"Extension no permitida: {ext}"
                )
            if mime not in allowed_mimes:
                raise HTTPException(
                    status_code=status.HTTP_400_BAD_REQUEST,
                    detail=f"Mime type no permitido: {mime}"
                )

        max_size = settings.MAX_FILE_SIZE_MB * 1024 * 1024
        if size_bytes > max_size:
            raise HTTPException(
                status_code=status.HTTP_400_BAD_REQUEST,
                detail=f"El archivo supera el maximo permitido de {settings.MAX_FILE_SIZE_MB} MB"
            )

    @staticmethod
    def save_professional_document(
        *,
        professional_id: str,
        document_type: str,
        upload: UploadFile,
        file_bytes: bytes,
        force_pdf: bool = False,
    ) -> Tuple[str, str, str, int]:
        LocalFileStorageService._validate_file(upload, len(file_bytes), force_pdf=force_pdf)

        base = LocalFileStorageService._ensure_base_path()
        sha256 = hashlib.sha256(file_bytes).hexdigest()
        ext = Path(upload.filename or "").suffix.lower()
        internal_name = f"{uuid.uuid4().hex}{ext}"

        relative_dir = Path("professional_documents") / professional_id / document_type.lower()
        absolute_dir = base / relative_dir
        absolute_dir.mkdir(parents=True, exist_ok=True)

        absolute_path = absolute_dir / internal_name
        with open(absolute_path, "wb") as f:
            f.write(file_bytes)

        relative_path = str((relative_dir / internal_name).as_posix())
        return relative_path, sha256, upload.content_type or "application/octet-stream", len(file_bytes)

    @staticmethod
    def get_absolute_path(relative_path: str) -> Path:
        base = LocalFileStorageService._ensure_base_path()
        candidate = (base / relative_path).resolve()
        if not str(candidate).startswith(str(base)):
            raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail="Ruta de archivo invalida")
        return candidate

    @staticmethod
    def soft_delete_file(relative_path: str) -> None:
        source = LocalFileStorageService.get_absolute_path(relative_path)
        if not source.exists():
            return

        deleted_dir = source.parent / "_deleted"
        deleted_dir.mkdir(parents=True, exist_ok=True)
        target = deleted_dir / f"{uuid.uuid4().hex}_{source.name}"
        shutil.move(str(source), str(target))
```

from pathlib import Path

from fastapi import APIRouter, Depends, HTTPException, status
from fastapi.responses import FileResponse
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession

from app.core.database import get_db
from app.core.dependencies import get_current_user
from app.models.file import File
from app.models.professional import Professional
from app.models.role import RoleCode
from app.models.user import User

router = APIRouter()


async def _can_access_file(file_row: File, current_user: User, db: AsyncSession) -> bool:
    await current_user.awaitable_attrs.roles
    role_codes = {role.code for role in current_user.roles}

    if RoleCode.SUPER_ADMIN.value in role_codes or RoleCode.ADMIN_VALIDATION.value in role_codes:
        return True

    if file_row.owner_user_id and str(file_row.owner_user_id) == str(current_user.id):
        return True

    prof_result = await db.execute(
        select(Professional).where(Professional.user_id == current_user.id)
    )
    professional = prof_result.scalar_one_or_none()
    if professional and str(file_row.owner_user_id) == str(current_user.id):
        return True

    return False


@router.get("/{file_id}/meta")
async def get_file_meta(
    file_id: str,
    current_user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
):
    result = await db.execute(
        select(File).where(File.id == file_id, File.deleted_at.is_(None))
    )
    file_row = result.scalar_one_or_none()
    if not file_row:
        raise HTTPException(status_code=404, detail="Archivo no encontrado")

    if not await _can_access_file(file_row, current_user, db):
        raise HTTPException(status_code=status.HTTP_403_FORBIDDEN, detail="Sin permisos para este archivo")

    return {
        "id": file_row.id,
        "original_filename": file_row.original_filename,
        "mime_type": file_row.mime_type,
        "size_bytes": int(file_row.size_bytes),
        "sha256": file_row.sha256,
        "relative_path": file_row.relative_path,
        "access_level": str(file_row.access_level.value if file_row.access_level else "private"),
        "owner_user_id": file_row.owner_user_id,
    }


@router.get("/{file_id}/download")
async def download_file(
    file_id: str,
    current_user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
):
    result = await db.execute(
        select(File).where(File.id == file_id, File.deleted_at.is_(None))
    )
    file_row = result.scalar_one_or_none()
    if not file_row:
        raise HTTPException(status_code=404, detail="Archivo no encontrado")

    if not await _can_access_file(file_row, current_user, db):
        raise HTTPException(status_code=status.HTTP_403_FORBIDDEN, detail="Sin permisos para descargar este archivo")

    absolute_path = LocalFileStorageService.get_absolute_path(file_row.relative_path)
    if not absolute_path.exists():
        raise HTTPException(status_code=404, detail="El archivo fisico no existe en disco")

    return FileResponse(
        path=str(absolute_path),
        media_type=file_row.mime_type,
        filename=file_row.original_filename,
    )

@router.post("/me/documents")
async def upload_document(
    document_type: DocumentType = Form(...),
    file: UploadFile = FastAPIFile(...),
    current_user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
):
    professional = await _get_current_professional(current_user, db)

    file_bytes = await file.read()
    force_pdf = document_type in {
        DocumentType.DEGREE,
        DocumentType.REGISTRATION_CERTIFICATE,
        DocumentType.SIGNED_AGREEMENT,
    }

    relative_path, sha256_hash, mime_type, size_bytes = LocalFileStorageService.save_professional_document(
        professional_id=str(professional.id),
        document_type=document_type.value,
        upload=file,
        file_bytes=file_bytes,
        force_pdf=force_pdf,
    )

    existing_result = await db.execute(
        select(ProfessionalDocument).where(
            ProfessionalDocument.professional_id == professional.id,
            ProfessionalDocument.document_type == document_type,
            ProfessionalDocument.deleted_at.is_(None),
        )
    )
    existing_docs = existing_result.scalars().all()

    for old_doc in existing_docs:
        old_doc.deleted_at = datetime.utcnow()
        old_doc.deleted_by = current_user.id

        old_file_result = await db.execute(
            select(File).where(File.id == old_doc.file_id, File.deleted_at.is_(None))
        )
        old_file = old_file_result.scalar_one_or_none()
        if old_file:
            old_file.deleted_at = datetime.utcnow()
            old_file.deleted_by = current_user.id
            LocalFileStorageService.soft_delete_file(old_file.relative_path)

    stored_file = File(
        relative_path=relative_path,
        original_filename=file.filename or "archivo.pdf",
        mime_type=mime_type,
        size_bytes=str(size_bytes),
        sha256=sha256_hash,
        is_encrypted="false",
        access_level=AccessLevel.SENSITIVE,
        owner_user_id=current_user.id,
    )
    db.add(stored_file)
    await db.flush()

    doc = ProfessionalDocument(
        professional_id=professional.id,
        document_type=document_type,
        file_id=stored_file.id,
        original_filename=file.filename or "archivo.pdf",
        mime_type=mime_type,
        sha256=sha256_hash,
        review_status=ReviewStatus.PENDING,

        updated_by=current_user.id,

        professional.is_public_profile_enabled = False

        db,
        action="professional_document_uploaded",
        entity_type="professional_document",
        entity_id="pending",
        actor_user_id=str(current_user.id),
        after_json=_json({
            "professional_id": professional.id,
            "document_type": document_type.value,
            "original_filename": file.filename,
            "file_id": stored_file.id,
            "sha256": sha256_hash,
        }),
    )

    await db.commit()
    await db.refresh(doc)

    return {
        "id": doc.id,
        "professional_id": doc.professional_id,
        "document_type": doc.document_type.value,
        "file_id": doc.file_id,
        "original_filename": doc.original_filename,

        "review_status": doc.review_status.value,

        .order_by(VerificationRequest.created_at.desc())
    )
    latest_request = latest_request_result.scalars().first()

    if latest_request and latest_request.status in {
        VerificationRequestStatus.SUBMITTED,
        VerificationRequestStatus.UNDER_REVIEW,
    }:
        raise HTTPException(
            status_code=status.HTTP_409_CONFLICT,
            detail="Ya existe una solicitud de validacion en curso"
        )

    verification = VerificationRequest(
        professional_id=professional.id,
        submitted_at=datetime.utcnow(),
        status=VerificationRequestStatus.SUBMITTED,
        created_by=current_user.id,
        updated_by=current_user.id,
    )
    db.add(verification)
    await db.flush()

    db.add(
        VerificationEvent(
            verification_request_id=verification.id,
            event_type=VerificationEventType.SUBMITTED,
            event_payload_json=_json({
                "documents_count": len(docs),
                "has_degree_pdf": has_degree_pdf,
                "source": "professional_self",
            }),
            created_by=current_user.id,
        )
    )

    professional.onboarding_status = OnboardingStatus.SUBMITTED
    professional.status = ProfessionalStatus.PENDING_REVIEW
    professional.is_public_profile_enabled = False

    await _register_audit(
        db,
        action="verification_submitted",
        entity_type="verification_request",
        entity_id=verification.id,
        actor_user_id=str(current_user.id),
        after_json=_json({

export type ApiFetchOptions = {
  method?: 'GET' | 'POST' | 'PUT' | 'PATCH' | 'DELETE'
  body?: unknown
}

export function useApi() {
  const config = useRuntimeConfig()
  const token = useCookie<string | null>('access_token')

  async function apiFetch<T>(path: string, options: ApiFetchOptions = {}): Promise<T> {
    const headers: Record<string, string> = {}

    const isFormData = typeof FormData !== 'undefined' && options.body instanceof FormData

    if (!isFormData) {
      headers['Content-Type'] = 'application/json'
    }

    if (token.value) {
      headers.Authorization = `Bearer ${token.value}`
    }

    return await $fetch<T>(`${config.public.apiBase}${path}`, {
      method: options.method ?? 'GET',
      headers,
      body: options.body as BodyInit | Record<string, unknown> | undefined,
    })
  }

  return { apiFetch }
}

## 5) Flujo funcional esperado

1. El médico entra con su cuenta.
2. Va a **/professional/profile**.
3. Completa perfil mínimo.
4. Sube el **PDF del título** con tipo `degree`.
5. Envía solicitud de validación.
6. El administrador entra a **/admin/validation/requests**.
7. Descarga el PDF.
8. Valida manualmente contra Senescyt fuera del sistema.
9. Aprueba o rechaza el documento.
10. Aprueba o rechaza la solicitud.
11. Solo si la solicitud queda aprobada, el médico pasa a `ACTIVE` y `is_public_profile_enabled = true`.

Ese comportamiento está alineado con la regla funcional definida en el proyecto: un profesional no debe quedar público ni activo sin documentos obligatorios y sin solicitud de verificación aprobada.

## 6) Pruebas de verificación y regresión

### Pruebas nuevas

1. Registrar un profesional nuevo.
2. Iniciar sesión como profesional.
3. Entrar a `/professional/profile`.
4. Guardar perfil mínimo.
5. Subir un PDF válido como `degree`.
6. Verificar que el documento aparezca listado.
7. Enviar solicitud de validación.
8. Iniciar sesión como `super_admin` o `admin_validation`.
9. Ir a `/admin/validation/requests`.
10. Abrir el detalle.
11. Descargar el archivo.
12. Aprobar el documento.
13. Aprobar la solicitud.
14. Volver a consultar `/api/v1/professionals/me` con la cuenta del médico y confirmar:

* `status = active`
* `onboarding_status = approved`
* `is_public_profile_enabled = true`

### Regresión obligatoria de “no rotura”

Esto también debe seguir funcionando igual:

* login con `/auth/login`; 
* sesión persistente del frontend;
* agenda del profesional en `/professional/agenda`, que ya consume `/professionals/me`, `/professionals/me/availabilities` y `/professionals/me/time-blocks`;
* dashboard admin;
* listado de citas;
* búsqueda pública de profesionales;
* Docker local levantando backend, frontend y postgres.

### Prueba puntual de archivos

Dentro del contenedor backend:

```bash
docker compose exec backend_api sh
ls -R /app/files/professional_documents
```

Debe verse el árbol de almacenamiento físico del PDF.

## 8) Observación técnica importante para el programador

No se debe intentar integrar una consulta automática a Senescyt en este paso. En esta fase solo se debe dejar el flujo **manual y auditable**: el médico sube el PDF, el administrador lo descarga y valida externamente, y recién después aprueba o rechaza dentro del sistema. Eso minimiza roturas, evita depender de integraciones inciertas y cumple el objetivo funcional que se pidió.

El contexto del repositorio usado para esta propuesta está en el repomix subido por el usuario.
