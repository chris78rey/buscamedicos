ACTUAR COMO DESARROLLADOR BACKEND SENIOR Y MODIFICAR SOLO EL SUBSISTEMA DE ARCHIVOS PARA DOCUMENTOS DEL PROFESIONAL.

ARCHIVOS A TOCAR
- backend_api/app/services/file_storage_service.py   (crear)
- backend_api/app/routers/files.py                   (modificar)
- backend_api/app/core/config.py                     (modificar si hace falta)

OBJETIVO
Dejar persistencia física real de documentos y descarga real desde backend.

CONTEXTO ACTUAL
El proyecto ya tiene:
- modelo File
- tabla files
- volumen Docker /app/files
- FILES_PATH en configuración
- router files existente
Pero el flujo del profesional no lo usa correctamente todavía.

TRABAJO EXACTO

A. Crear servicio nuevo
Crear:
- backend_api/app/services/file_storage_service.py

Ese servicio debe:
1. leer FILES_PATH desde settings
2. crear carpeta base si no existe
3. validar:
   - extensión permitida
   - mime type permitido
   - tamaño máximo
4. guardar archivo en:
   /app/files/professional_documents/{professional_id}/{document_type}/{uuid}.pdf
5. calcular sha256
6. devolver:
   - relative_path
   - sha256
   - mime_type
   - size_bytes
7. exponer helper para:
   - get_absolute_path(relative_path)
   - soft_delete_file(relative_path)
     moviéndolo a carpeta _deleted

B. Ajustar config
En backend_api/app/core/config.py:
- asegurarse de que FILES_PATH exista y por defecto sea /app/files
- agregar máximo tamaño configurable
- agregar listas permitidas de mime y extensión

C. Ajustar files.py
El router files debe permitir:
1. GET /api/v1/files/{file_id}/meta
2. GET /api/v1/files/{file_id}/download

Reglas:
- solo puede descargar:
  - el dueño del archivo
  - super_admin
  - admin_validation
- si el archivo físico no existe, debe devolver 404
- si no tiene permisos, debe devolver 403

D. Integración con professional_self.py
La instrucción 1 debe llamar este servicio.
No se debe duplicar lógica de guardado dentro del router.

DETALLE IMPORTANTE
Cuando se suba un documento de tipo degree:
- debe forzar application/pdf
- extensión .pdf

NO HACER
- No meter S3.
- No meter MinIO.
- No meter almacenamiento externo.
- No borrar archivos físicamente.
- No rehacer modelos.

PRUEBAS DE ESTE BLOQUE
1. levantar Docker
2. subir documento de tipo degree
3. revisar dentro del contenedor:
   docker compose exec backend_api sh
   ls -R /app/files/professional_documents
4. consultar:
   GET /api/v1/files/{file_id}/meta
5. descargar:
   GET /api/v1/files/{file_id}/download
6. eliminar documento desde endpoint profesional
7. verificar que el archivo pase a _deleted

CRITERIO DE ACEPTACIÓN
- el archivo queda guardado físicamente,
- existe registro real en files,
- se puede descargar,
- al eliminar no se pierde definitivamente.
